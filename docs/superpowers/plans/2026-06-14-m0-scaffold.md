# M0 — medusa-cz Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the `medusa-cz` Turborepo + pnpm monorepo — tooling, CI, the thin `@medusa-cz/shared` package (proven via TDD), four empty-but-buildable plugin skeletons, and a demo Medusa app that consumes a workspace package — with zero plugin business logic.

**Architecture:** pnpm workspaces + Turborepo task graph. Four publishable Medusa-2.15 plugin packages (`payment-comgate`, `fulfillment-packeta`, `payment-gopay`, `invoicing-fakturoid`) each built with `medusa plugin:build`; one plain-TS library `@medusa-cz/shared` (open-core seam) built with `tsup`; one `apps/demo` Medusa backend (+ Next.js storefront from `create-medusa-app`) that symlinks the workspace packages. Independent per-package publish via Changesets. CI runs typecheck/lint/test/build (Turbo-cached) + DCO sign-off check.

**Tech Stack:** pnpm, Turborepo, TypeScript 5.6+, MedusaJS **2.15.5**, `@medusajs/test-utils` (Jest) for integration, Vitest for unit, tsup, ESLint 9 (flat) + Prettier, Changesets, GitHub Actions.

**Authoritative facts:** All Medusa-2.15 specifics in this plan are verified in `docs/superpowers/research/2026-06-14-medusa-2.15-verification.md` (the §6 gate). If a step conflicts with that file, that file wins.

**Scope guardrails (from spec §9 / §10):** M0 is scaffold only. NO provider/module business logic — skeletons must compile and build but implement nothing. Comgate/Packeta/GoPay/Fakturoid logic is M1–M4. iÚčto/Balíkobot/iDoklad/POHODA are out of scope.

**Environment prereqs (verify before Task 14):** Node ≥20 (≤24 for storefront), `corepack enable`, a reachable PostgreSQL instance for the demo + integration tests, git configured.

## Task 8: TDD the first `shared` utility — Czech IČO validator

This task proves the full RED→GREEN→test chain end to end. The IČO checksum is a real, deterministic CZ value rule (spec §8).

**Files:**

- Create: `packages/shared/src/cz/ico.ts`, `packages/shared/src/cz/ico.test.ts`

- [ ] **Step 1: Write the failing test** — create `packages/shared/src/cz/ico.test.ts`

```ts
import { describe, expect, it } from "vitest"
import { isValidIco } from "./ico"

describe("isValidIco", () => {
  it("accepts a valid 8-digit IČO (checksum ok)", () => {
    // 49619331: weighted sum 187, 187 % 11 == 0 -> check digit 1 -> matches
    expect(isValidIco("49619331")).toBe(true)
  })

  it("accepts a valid IČO with leading zeros", () => {
    // 00006947 is a real valid IČO (Czech National Bank); checksum holds with leading zeros
    expect(isValidIco("00006947")).toBe(true)
  })

  it("rejects a number whose check digit is wrong", () => {
    expect(isValidIco("49619332")).toBe(false)
  })

  it("rejects wrong length", () => {
    expect(isValidIco("1234567")).toBe(false)
    expect(isValidIco("123456789")).toBe(false)
  })

  it("rejects non-numeric input", () => {
    expect(isValidIco("4961933X")).toBe(false)
    expect(isValidIco("")).toBe(false)
  })

  it("tolerates surrounding whitespace", () => {
    expect(isValidIco("  49619331 ")).toBe(true)
  })
})
```

- [ ] **Step 3: Write the minimal implementation** — create `packages/shared/src/cz/ico.ts`

```ts
/**
 * Validates a Czech IČO (company registration number).
 *
 * Rule: exactly 8 digits. The first 7 digits are weighted 8,7,6,5,4,3,2;
 * the check digit is `(11 - (weightedSum % 11)) % 10` and must equal the 8th digit.
 * Leading zeros are significant (IČO is an identifier string, not a number).
 */
export function isValidIco(input: string): boolean {
  const ico = input.trim()
  if (!/^\d{8}$/.test(ico)) {
    return false
  }

  const digits = ico.split("").map((c) => Number(c))
  let weightedSum = 0
  for (let i = 0; i < 7; i++) {
    weightedSum += digits[i]! * (8 - i)
  }

  const checkDigit = (11 - (weightedSum % 11)) % 10
  return checkDigit === digits[7]
}
```

## Task 16: CI workflow — typecheck, lint, test, build, DCO

**Files:**

- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  dco:
    name: DCO sign-off check
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Verify Signed-off-by on all PR commits
        run: |
          set -e
          base="${{ github.event.pull_request.base.sha }}"
          head="${{ github.event.pull_request.head.sha }}"
```
